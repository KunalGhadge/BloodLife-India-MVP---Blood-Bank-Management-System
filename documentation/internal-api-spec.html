17. How Different Parts of the App Talk to Each Other

Even though we don't have an external API yet, our internal modules follow a strict "Contract." This ensures that when the Dashboard asks for the inventory, it gets exactly what it expects, every single time.

Tech Tip: TypeScript Interfaces. These act as a "dictionary" for our code, defining what a Donor or a BloodUnit looks like.

Our Main Logic Modules

  • Match Management: Functions like addMatch() and updateMatchStatus() handle the relationship between donors and requests.
  • Inventory Hub: The central place for addStock() and checkExpiry() calls.
  • Storage Wrapper: A simplified set of commands to read and write to the local database.

Standard Format

All our internal "API" calls return predictable results. If an operation fails, the function returns a clear error message instead of just "crashing," allowing the UI to show a helpful alert to the user.

Why a Spec Matters?

A clear specification makes it easy for multiple developers to work on the same project without stepping on each other's toes. If you know exactly what input a function needs, you can build your feature with confidence.
