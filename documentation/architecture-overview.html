04. Understanding the System Architecture

BloodLife India is built as a Local-First Web Application. This means the app runs entirely in the user's browser without needing a dedicated backend server for its basic functions. This makes it incredibly fast, private, and reliable in areas with poor internet connection.

Key Concept: Client-Dominant Architecture. The browser does all the "heavy lifting"—from matching blood groups to saving data.

The Three Main Layers

  • The Visual Interface (UI): Built with React 19 and Tailwind CSS. It's designed to be clean, responsive, and easy to use on both mobile and desktop.
  • The Core Logic (The Brain): Written in TypeScript. This layer handles the donor matching algorithms and manages the application "state" (what's currently happening in the app).
  • The Storage Layer: Uses the browser's localStorage to keep your data safe. Think of it as a small database that lives right inside your Chrome or Firefox browser.

Data Flow

When you add a new donor, the following happens:

  1. The UI captures the data and sends it to the Core Logic.
  2. The Core Logic validates the details and updates the app's current memory.
  3. The Storage Layer immediately saves a copy so that your data is still there when you refresh the page.
