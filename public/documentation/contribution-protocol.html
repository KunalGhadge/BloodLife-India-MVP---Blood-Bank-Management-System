03. How to Contribute to BloodLife

We believe in the power of the community! Whether you're fixing a bug, adding a new feature, or improving the documentation, your help is welcome. Follow these steps to ensure a smooth contribution process.

Pro Tip: We use TypeScript for everything. Please ensure your code passes all type checks before submitting.

Our Coding Standards

To keep the project clean and maintainable, please follow these rules:

  • Be Explanatory: Use clear variable names like donorList instead of dl.
  • Stay Consistent: We use Tailwind CSS for styling. Avoid writing custom CSS in separate files unless necessary.
  • Test Your Changes: Run the app locally and verify that your changes don't break existing features like donor matching or inventory tracking.

The Contribution Process

  1. Fork the Repo: Create a copy of the project on your GitHub account.
  2. Create a Branch: Work on a new branch (e.g., feature/add-dark-mode).
  3. Commit Often: Make small, descriptive commits (e.g., fix: resolve donor search bug).
  4. Push and Pull: Push your changes to your fork and open a "Pull Request" to the main repo.

Need Help?

If you're stuck, feel free to open an issue on the GitHub repository. We are here to help you get started!
