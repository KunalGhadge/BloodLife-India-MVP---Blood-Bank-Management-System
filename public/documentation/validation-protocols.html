13. Ensuring Data Integrity

In a healthcare-related application, "bad data" can have serious consequences. Our validation system ensures that every piece of information entered into BloodLife is accurate, formatted correctly, and safe to use.

Pro Concept: Schema Validation. We define a set of rules (a "schema") that data must follow before it's saved.

Types of Validation

  • Format Checks: Ensures email addresses look like emails, and phone numbers contain valid digits.
  • Medical Validation: Keeps blood groups restricted to the valid types (e.g., A+, AB-, O+).
  • Mandatory Fields: Prevents "empty" donors or requests from being saved.

Real-Time Feedback

The system validates data *as you type*. If you make a mistake (like entering a future date for a donor's birthday), the system immediately highlights the error and explains how to fix it.

TypeScript Protection

During development, TypeScript acts as a static shield. It prevents developers from accidentally passing the wrong type of data (like sending a "Number" where a "Name" should be) at the code level.
