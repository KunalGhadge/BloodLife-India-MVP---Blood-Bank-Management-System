20. Ensuring Quality and Reliability

Before any piece of code is released, it goes through a rigorous testing process. We want to make sure that the "MatchFinder" never fails and that donor data is always handled correctly.

Our Mantra: Zero Tolerance for Bugs. In a system that helps save lives, technical reliability is an ethical requirement.

How We Verify Our App

  • Static Analysis: We use ESLint and TypeScript to catch errors *while we are coding*, before the app is even run.
  • Manual Walkthroughs: Our team performs "End-to-End" tests—simulating a user creating a request, finding a donor, and completing a match.
  • Edge Case Testing: We deliberately try to break the app by entering weird data (like negative unit counts) to see if the validation layers catch it.

Continuous Integration (CI)

Every time a developer pushes code to GitHub, an automated script runs to check if the project still "builds" successfully. If there's a problem, the script stops the process before the broken code can reach any users.

User Feedback Loop

We rely on our community of users to report anomalies. If something feels slow or doesn't work as expected, we treat it as a top priority for our next verification cycle.
