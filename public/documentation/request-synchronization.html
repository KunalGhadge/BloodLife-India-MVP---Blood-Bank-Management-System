09. Handling Blood Requests

The Request Management system handles the urgent needs of hospitals and patients. It’s designed to ensure that no request goes unnoticed and that the most critical cases are handled first.

Key Mechanism: Urgency Scaling. We categorize requests based on how quickly the blood is needed (e.g., Emergency vs. Routine).

Creating a Request

A request includes several critical pieces of information:

  • Recipient Information: Who needs the blood and what is their blood type?
  • Quantity: How many units are required?
  • Urgency: Is this an immediate emergency (critical) or can it wait (routine)?
  • Location: Where should the blood be delivered?

The Fulfillment Queue

Once a request is created, it enters the system's "Request List." From here, admins can:

  1. Find Matches: Use the MatchFinder to see compatible donors.
  2. Assign Units: Link units from the Inventory to specific requests.
  3. Track Status: Mark requests as "Pending," "Partially Fulfilled," or "Completed."

Synchronizing Requests

Since BloodLife is local-first, we use a sophisticated state manager to ensure that if multiple people are working on the same device, the request list stays up-to-date and consistent.
